# VK Link Rewriter (Python)

Инструмент для массовой замены ссылок в постах и комментариях ВК в ваших пабликах.

**Репозиторий:** [github.com/zakcheks/vk_public_rewriter](https://github.com/zakcheks/vk_public_rewriter)

## Установка

1. Установите Python 3.9+.
2. В каталоге проекта выполните:


python3 -m venv venv
source venv/bin/activate
```bash
pip install -r requirements.txt
```

## Токен ВК

Нужен пользовательский `access_token` с правами:

- `wall`
- `groups`
- `offline`

Его можно получить через официальное приложение/standalone-приложение ВК.

Рекомендуемый вариант хранения:

1. Создайте файл `.env` рядом со скриптом:

```text
VK_TOKEN=ВАШ_ACCESS_TOKEN
```

2. Либо при запуске просто введите токен в консоль/окне, когда его попросит программа.

## Запуск

### Вариант 1. Консольный режим

```bash
python vk_link_rewriter.py
```

Далее скрипт спросит в терминале:

1. **VK_TOKEN**, если он не задан в `.env`.
2. **Старую ссылку** — то, что нужно найти в тексте постов/комментариев.
3. **Новую ссылку** — то, на что заменить.
4. **Список пабликов** — по одному в строке:
   - `https://vk.com/public123`
   - `https://vk.com/club123`
   - `public123`, `club123`
   - `https://vk.com/имя_сообщества`

Пустая строка завершает ввод списка пабликов.

### Вариант 2. Графический интерфейс (PyQt6)

```bash
python vk_link_gui.py
```

В появившемся окне укажите:

- **VK токен** — можно оставить пустым, если он задан в `.env` как `VK_TOKEN`.
- **Старую ссылку** — что искать в постах/комментариях.
- **Новую ссылку** — на что заменить.
- **Ссылки на сообщества** — по одному паблику в строке.

Нажмите кнопку **«Запустить»**. Лог работы и все сообщения будут отображаться в нижней части окна.  
Кнопкой **«Остановить»** можно прервать текущую обработку.

### Вариант 3. Веб-интерфейс

Одна страница с формой и логом. При нажатии **«Запустить»** запускается скрипт `vk_link_rewriter.py` (без отдельного API и без Vue/npm).

```bash
python app.py
```

Откройте **http://127.0.0.1:5000**. Заполните форму (VK токен, старая/новая ссылки, список сообществ по одному в строке) и нажмите **«Запустить»** — скрипт выполнится, лог обновляется в реальном времени. **«Остановить»** прерывает задачу между сообществами.

На главной странице отображается **текущий VK-аккаунт** по введённому токену (запрос к `/api/me`). При запуске замены в лог также выводится аккаунт, от имени которого выполняется операция.

### Деплой (GitHub Actions)

Workflow `.github/workflows/deploy.yml` деплоит приложение на VPS по SSH при пуше в `main` или по кнопке «Run workflow».

**Секреты репозитория** — добавьте в [Settings → Secrets and variables → Actions](https://github.com/zakcheks/vk_public_rewriter/settings/secrets/actions):

| Секрет        | Описание                              |
|---------------|----------------------------------------|
| `DEPLOY_HOST` | IP или домен сервера                   |
| `DEPLOY_USER` | Пользователь SSH                      |
| `DEPLOY_KEY`  | Приватный SSH-ключ (содержимое файла) |
| `DEPLOY_PATH` | Каталог приложения на сервере         |

**На сервере один раз:** клонировать [репозиторий](https://github.com/zakcheks/vk_public_rewriter) в `DEPLOY_PATH`, при необходимости настроить systemd-сервис `vk-rewriter` для gunicorn (иначе используется запуск через `nohup`).

## Админ-панель (веб)

Для аудита действий VK-аккаунтов и блокировки доступа по VK User ID доступна админ-панель.

1. В `.env` задайте:
   - `ADMIN_PASSWORD` — пароль входа в админку (если не задан, панель отключена).
   - `FLASK_SECRET_KEY` — секрет для сессий (обязательно в продакшене).
   - При необходимости `ADMIN_DB_PATH` — путь к SQLite-файлу (по умолчанию `admin_data.sqlite` в каталоге проекта).

2. Запустите приложение и откройте админ-панель по адресу **/g4t54t45yytg5tyg5yt** (например, http://127.0.0.1:5000/g4t54t45yytg5tyg5yt) — страница входа. После входа доступны:
   - **Аудит** — кто (VK ID, имя), когда и с какого IP запускал замену ссылок.
   - **Аудит посещений админ-панели** — по IP: страница входа, дашборд, успешный/неудачный вход, выход.
   - **Смена пароля** — смена пароля входа в админку (текущий + новый пароль).
   - **Заблокированные** — список заблокированных VK User ID с возможностью разблокировки.
   - **Блокировка** — ввод VK User ID (и опционально причины) для блокировки аккаунта.

Заблокированные аккаунты при запросе `/api/run` получают ответ **403** с сообщением о блокировке.
